Layer Architecture

Core Layer

HeadlessRunner — Creates an Engine with a viewport (480×720), runs the game loop for N frames, returns a SimResult. Game-agnostic: takes function pointers for setup/update/render.

SimResult — Universal output: frames_run, game_state (HashMap), framebuffer_hash (FNV-1a), elapsed_sim_time. New: get_f64() convenience accessor.

ScheduledAction — Timed input events: PointerDown, PointerMove, PointerUp at specific frames.

GameScenario / ScenarioBuilder — Declarative tests with assertions. Builder reduces boilerplate for multiple scenarios.

ShotBuilder — High-level slingshot input builder. Angle + power → drag vector.

Analysis Layer

Parameter Sweep — run_sweep() runs the same scenario with different state overrides. SweepReport supports min_by_state(), max_by_state(), and summaries.

State Timeline — record_timeline() captures specific keys at every frame. Query with series(), stats(), first_frame_where().

Fitness Evaluator — Composable weighted scoring. Each criterion maps SimResult → [0, 1]. Produces letter grades (A+ through F) and can rank sweeps.

Regression Suite — Capture baselines, diff future runs. Pluggable classifiers for domain-aware improvement/regression detection.

Snapshot & Replay Layer

Snapshots — run_with_snapshots() captures full game state at specific frames. Best for debugging specific moments.

Replay — record_replay() captures tracked keys at every frame. Lightweight. Supports series(), get(), first_frame_where().

Comparison — compare_replays() produces per-key max_delta, mean_delta, first_divergence, and visual divergence detection.

Anomaly Detection — AnomalyDetector scans for spikes (sudden jumps), plateaus (stuck values), and out-of-bounds values.

Optimization Layer

Experiment — Combines sweep + fitness + optional regression into Experiment::new().with_*().run(). Returns rankings, verdicts, summaries.

Hill Climber — Coordinate-descent optimizer. Given ParamRange definitions, tries ±step for each parameter. Shrinks step when stuck. Returns best params and convergence history.

Action Generator — Programmatic input: grid_shots() for systematic exploration, random_shots() for PRNG-based coverage, tap_sequence() for UI testing, drag() for gestures.

Orchestration Layer

Strategy — Multi-step playbooks: chain Record → Compare → DetectAnomalies → AssertState steps. Each step operates on named replays.

Test Harness — Battery testing: multiple scenarios with optional fitness. Produces HarnessReport with pass/fail counts and average fitness.

Golden Test — Reference replay comparison. Record golden when correct, diff future runs. The headless equivalent of visual regression testing.

Data Flow: A Typical AI Iteration

1. Claude reads game code, identifies physics parameters
2. Creates SweepConfig[] with parameter variations
3. Runs Experiment with fitness evaluator
4. Inspects ExperimentResult.rankings — finds best config
5. Uses HillClimber to fine-tune the best parameters
6. Records golden replay of improved behavior
7. Modifies game code with optimized values
8. Runs GoldenTest to verify no regressions
9. Runs TestHarness for comprehensive quality check
10. Commits with structured test evidence

Game Integration

Every headless module is game-agnostic. Games integrate by providing 4 function pointers:

fn setup(engine: &mut Engine)                              // Initialize state
fn update(engine: &mut Engine, dt: f64)                     // Game logic per frame
fn render(engine: &mut Engine)                              // Draw to framebuffer
fn action_dispatch(engine: &mut Engine, action: &ScheduledAction)  // Route inputs

The S-League demo provides: sleague::setup_fight_only, sleague::update, sleague::render, sleague::dispatch_action.
