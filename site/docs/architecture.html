Headless Testing Architecture

22-module infrastructure for automated game testing, optimization, and analysis — all without a browser, display, or human intervention. Runs via cargo test and the engine-cli tool.

Core Abstractions

Simulation Trait

The contract between a game and the engine:


pub trait Simulation {
    fn setup(&mut self, engine: &mut Engine);
    fn step(&mut self, engine: &mut Engine);
    fn render(&self, engine: &mut Engine);
    fn variants(&self) -> Vec<ParamSet> { vec![] }
}

InputFrame

Canonical representation of one frame of player input. Serializable for replays.


pub struct InputFrame {
    pub keys_pressed: Vec<String>,
    pub keys_released: Vec<String>,
    pub keys_held: Vec<String>,
    pub pointer: Option<(f64, f64)>,
    pub pointer_down: Option<(f64, f64)>,
    pub pointer_up: Option<(f64, f64)>,
}

Policy Trait

Pluggable input generator. Takes an Observation, produces an InputFrame:


pub trait Policy {
    fn next_input(&mut self, obs: &Observation) -> InputFrame;
}

Built-in: NullPolicy (no input), RandomPolicy (seeded random), ScriptedPolicy (replay fixed inputs).

Observation

Lightweight view into engine state after each frame:


pub struct Observation<'a> {
    pub frame: u64,
    pub state_hash: u64,
    pub game_state: &'a GameState,
    pub entity_count: usize,
    pub framebuffer: Option<&'a [u8]>,
}

Core Layer

HeadlessRunner

The foundation. Creates an Engine with a specified viewport, drives simulations, returns SimResult. Three execution modes: fixed inputs, policy-driven, and input-with-capture.

SimResult

Universal output: frames_run, game_state (HashMap), framebuffer_hash (FNV-1a), state_hash, state_hashes (optional per-frame), elapsed_sim_time.

PlaythroughFile

Serializable replay format: seed, inputs, frame count, state/framebuffer hashes, per-frame state hashes, metadata. verify() replays the same seed and inputs and compares per-frame hashes.

GameScenario

Declarative test scenarios with ScheduledAction (pointer inputs at specific frames) and Assertion (state equality, range, comparison, framebuffer checks). ScenarioBuilder reduces boilerplate for multi-scenario test suites.

Analysis Layer

Parameter Sweep

run_sweep() runs scenarios with different parameter configurations. Returns SweepReport queryable by min/max of any state key.

State Timeline

record_timeline() captures specific state keys at every frame. StateTimeline supports series(), stats(), and first_frame_where() queries.

Fitness Evaluator

Composable weighted scoring. Each criterion maps to [0, 1]. Produces FitnessResult with letter grades (A+ through F). Helpers: score_distance, score_state_match, score_ratio.

Regression Suite

Capture baselines, diff against future runs. Pluggable classifiers determine regression vs. improvement.

Replay Layer

Replay

record_replay() captures specified state keys at every frame. Lightweight and efficient for full-run analysis.

Comparison

compare_replays() produces structural diff: per-key max_delta, mean_delta, first_divergence, and full delta series.

Anomaly Detection

AnomalyDetector scans replay series for spikes, plateaus, and out-of-bounds values. Configurable thresholds.

Optimization Layer

Experiment

Combines sweep + fitness + optional regression into a single declarative call.

Hill Climber

Coordinate-descent optimizer. Tries +/-step for each parameter, keeps improvements, halves all steps when stuck. Convergence detection when steps fall below min_step. Returns fitness history for visualizing convergence.

Action Generator

Programmatic input generation: grid_shots(), random_shots(), tap_sequence(), drag().

Orchestration Layer

Strategy

Multi-step playbook: chain Record, Compare, DetectAnomalies, and AssertState steps.

Test Harness

Battery testing: run multiple scenarios, produce consolidated HarnessReport with pass/fail counts and average fitness.

Golden Test

Record a golden replay when behavior is correct, then diff future runs against it. Separated record_golden() and compare_against() workflow.

Design Acceleration Layer

Death Classification

Classifies terminal states by trajectory shape using pure math — slope and variance on the last N frames:

ClassPattern
CloseCallMetric was near threshold before terminal event
BlowoutMetric was far from threshold — never competitive
CliffSudden drop from healthy to terminal
AttritionSlow, steady decline to terminal

Interesting Moment Detection

Scans batch runs for notable events using rolling window statistics:

  • Spikes/Drops — deviation from rolling mean exceeding threshold * stddev
  • Near-death — drop from above threshold (prevents false positives on consistently low values)
  • Milestones — upward crossings of milestone values

Mechanic Ablation

Runs baseline vs. mechanic-disabled sweeps across multiple seeds. Computes mean scores, absolute deltas, and percentage changes. ranked() sorts mechanics by impact.

Divergence Replay

Compares two runs frame-by-frame via state hash comparison. Finds the exact divergence frame with configurable context window.

Variant Branching

Games declare parameter variants via Simulation::variants(). The engine sweeps all variants across seed ranges and compares outcomes.

Dashboard

Generates structured JSON combining sweep stats, death classification, highlights, and ablation results. All modules produce serde-serializable output.

Data Flow: A Typical AI Iteration


1. Create Simulation implementation
2. Run sweep across seed range with policy
3. Evaluate fitness, classify deaths
4. Use hill climber to fine-tune parameters
5. Record golden replay of improved behavior
6. Verify no regressions against golden baseline
7. Run highlights to find interesting moments
8. Commit with structured test evidence
