Engine Architecture

Crusty Engine is a deterministic simulation engine written in Rust that compiles to WebAssembly (browser) and native targets (headless testing, CLI). It features a custom ECS, software framebuffer renderer with SDF anti-aliasing, fixed-timestep physics with continuous collision detection, and seeded RNG for perfect reproducibility.

Simulation Trait

The formal boundary between engine and game logic. Games implement three methods:


pub trait Simulation {
    fn setup(&mut self, engine: &mut Engine);   // Initialize entities and state
    fn step(&mut self, engine: &mut Engine);    // Game logic every fixed tick (1/60s)
    fn render(&self, engine: &mut Engine);      // Draw to engine.framebuffer
    fn variants(&self) -> Vec<ParamSet> { vec![] }  // A/B test parameters
}

The engine owns timing, input application, RNG, and determinism. Games read engine.input and engine.global_state during step().

Wall-clock dt is clamped to 50ms (prevents spiral-of-death). The accumulator is further clamped to 5 * FIXED_DT to prevent runaway physics catch-up.

  • World — holds all entities + 32 component stores. spawn() / despawn() manage lifecycle.

Physics

Integration

Semi-implicit (symplectic) Euler. Velocity is updated from acceleration first, then the collision system applies the new velocity to compute position. This conserves energy in oscillatory systems (orbits, springs) much better than explicit Euler.

  • Frame-rate independent damping: (1.0 - damping).powf(dt)
  • ContinuousDrag with exponential decay: (-drag * dt).exp()

Collision Detection

Continuous Collision Detection (CCD) prevents tunneling:

  • Circle vs Circle — quadratic ray-sphere intersection on the Minkowski sum
  • Circle vs Line Segment — tests both sides of the line normal, checks segment bounds, falls back to endpoint sweeps
  • Circle vs AABB — decomposes into 4 edges + 4 corners

All three handle edge cases correctly: zero-length segments, stationary objects, already-overlapping objects, degenerate cases.

Collision Response

Iterative bounce loop with max_bounces = 4 handles multi-bounce scenarios (ball trapped in corners). remaining_t tracking ensures no simulation time is lost. Epsilon separation of 0.01 prevents surface sinking. Trigger zones are detected separately from physical collisions.

Spatial Grid

Uniform grid broadphase with pre-computed inv_cell_size. Cell size 128px. Rebuilt each frame. Deduplication via sort_unstable + dedup.

Force Accumulation

Clean zero-then-accumulate pattern. Two force falloff models:

  • Plummer softening — F(r) = strength * r / (r² + ε²)^(3/2). Standard N-body kernel, smooth at r=0, falls off as 1/r² at distance. Used for gravity wells.
  • InverseSquare with min-distance clamping to prevent singularities.

Zone effects: Wind, Drag, SpeedMultiplier, Conveyor.

Joints

Distance, spring, rope, and hinge constraints.

Rendering

Pure CPU software renderer. No GPU dependency. The framebuffer is a flat Vec<u8> in RGBA format.

Anti-Aliasing

All curved and diagonal primitives use signed distance field (SDF) evaluation with a 1-pixel feather zone. The pattern: pixels inside the shape get full alpha; pixels in the feather zone get alpha proportional to distance; pixels outside are skipped.

Text

Embedded 5×7 bitmap font covering ASCII 32-126. Zero external dependencies. Integer-multiple scaling. Functions: draw_text, draw_text_centered, text_width.

Particles

Hard cap of 2048 particles. Small particles (≤1.5px) render as single blended pixels for efficiency. retain_mut for single-pass update+removal. Deterministic via seeded RNG.

Post-Processing

  • Vignette — per-pixel distance-based edge darkening
  • CRT scanlines — periodic row dimming
  • Screen shake — sine-based offset with linear decay
  • ScreenFxStack — composable tint, desaturate (BT.601 luminance), flash effects with automatic expiration

Rendering Bridge

Two paths for browser display:

  • Canvas 2D (demo-ball, gravity-pong) — ImageData from WASM memory via framebuffer_ptr()
  • WebGL2 (chord-reps) — framebuffer uploaded as texture, multi-pass post-processing (bloom, Gaussian blur, chromatic aberration, scanlines, vignette)

Determinism

  • SeededRng (xorshift64) — the single canonical RNG, owned by Engine
  • Engine::state_hash() — deterministic u64 from simulation state, independent of rendering
  • Engine::reset(seed) — single entry point for reproducible simulation
  • Fixed DT (1/60s) for all simulation systems
  • InputFrame — canonical input representation for replays and policies
  • BTreeMap used throughout for deterministic iteration order

Same seed + same inputs = identical state hashes. Verified by the headless testing infrastructure across thousands of seeds.

WASM Integration

Zero-Copy Framebuffer

JS reads pixels directly from WASM linear memory via framebuffer_ptr(). No serialization, no copying on the Rust side.

BrowserState Shared Memory

Rather than calling back into WASM for browser metadata, JS writes a fixed Float64Array view into WASM linear memory. Rust reads typed slots by index. Zero-copy, zero-allocation per-frame data flow. Correctly handles WASM memory growth (buffer regrowth check).

Feature Gating

console_error_panic_hook and web-sys are properly cfg-gated under #[cfg(target_arch = "wasm32")]. The log.rs module abstracts over console::log (WASM) and println! (native).

Binary Size

Optimized with opt-level = "s", lto = true, codegen-units = 1. Minimal dependency set.

Game Support Systems

CameraDirector

Stack-based cinematic camera with 5 modes: Follow (entity tracking with dead zone), Pan (eased position), Zoom (eased scale), Shake (procedural sine-based), Letterbox (cinematic bars). Declarative rules trigger modes from EventBus events.

AutoJuice

Declarative game feel. 4 triggers (OnCollision, OnDespawn, OnSpawn, OnEvent) × 5 effects (particles, screen shake, screen flash, sound, freeze frame). Builder pattern for composing effects.

GameFlow

Declarative lifecycle FSM with states (Title, Playing, Paused, GameOver, Victory) and conditions (StateReaches, EventFired, ButtonTapped, AfterSeconds). First-matching-transition semantics.

FeelPreset

Named physics profiles. 6 built-in: tight_platformer, floaty_astronaut, heavy_tank, snappy_cursor, underwater, ice_skating. JSON/TOML serialization.

GestureRecognizer

Touch gesture recognition: Tap, DoubleTap, LongPress, Swipe (with direction + velocity), Pinch. Configurable thresholds. Primary touch forwarding for mouse compatibility.

Behavior Rules

Declarative condition-action rules. Conditions: Collision, TriggerEnter, Timer, StateCheck, KeyPress, Always. Actions: Despawn, SetState, SpawnTemplate, TimerControl, Log. One-shot support. Snapshot-then-commit pattern prevents ordering bugs.

State Machine

Per-entity FSM with one-frame edge detection (just_entered, prev_state). Transition conditions with duration and state checks. TimeScale integration for time-scaled entities.

Coroutines

Simulated async without Rust async. VecDeque<CoroutineStep> with front-popping. Non-blocking steps (SetState, Log, SpawnTemplate) cascade within the same frame. Wait steps yield.

Tween System

9 easing functions (Linear, EaseIn/Out/InOut, Back, Bounce, Elastic, Quad, Cubic). 7 tween targets (X, Y, Rotation, Scale, VelocityX, VelocityY, Alpha). Loop and ping-pong modes. TimeScale integration.

EventBus

Frame-scoped pub/sub. Dual string/typed channels. Builder pattern for event construction. Zero-allocation query API via iterators. Central integration point for cross-system communication.

Scene Manager

Stack-based push/pop semantics for nested scenes (overworld → dungeon → fight). Captures full World snapshots on push, restores on pop.

Telemetry

Delta-compressed state recording. Bitwise NaN comparison prevents spurious deltas. RFC 8259-compliant JSON escaping. Handles NaN, Infinity, and integer formatting.

Diagnostics

Runtime health checks: NaN transforms, out-of-bounds entities, entity count warnings.

Spatial & Procedural Systems

  • SpatialHashGrid — broad-phase with pre-computed inv_cell_size. nearest() queries.
  • Tilemap — multi-layer grid. Tile types: Empty, Solid, Platform, Custom(u16).
  • A* Pathfinding — octile heuristic, corner-cutting prevention, max-iteration safety valve.
  • Procedural Generation — value noise (smoothstep interpolation, multi-octave), cellular automata (birth/death thresholds), dungeon rooms (overlap detection, L-corridor connection). All deterministic via SeededRng.
  • DensityField, FlowNetwork, Raycast — specialized spatial query tools.

Games

Gravity Pong

A 10-level physics puzzle game — not a Pong clone. Players guide particles into targets using gravitational force fields.

  • Entity vocabulary: gravity wells (Plummer softening), repulsors, black holes (kill zones), wormholes (bidirectional teleporters), plasma currents (directional corridors), supernovas (timed explosives), walls (line-segment reflectors), waypoints (particle capture zones)
  • Core mechanic: tap to place waypoints that capture stopped particles, then sling-launch toward targets through gravitational fields
  • Visual juice: 300 field dust motes that respond to gravity (creating visible "field lines"), 50×50 gravitational field grid overlay, particle trails, screen shake, screen flash, expanding rings on supernova detonation
  • 10 levels with progressive introduction of mechanics: First Contact → The Sling → Orbital → Through the Wormhole → Current Affairs → Dark Side → Chaos Theory → The Gauntlet → Countdown → Final Frontier

Chord Reps

Interactive ear training game with spaced repetition.

  • 5 concept types: scale degrees, roman numerals, intervals, chord qualities, cadences (35 SRS cards total)
  • SM-2 SRS variant with custom modifications: hint-used gives 0.5-day interval (forgiving), fail gives 0.1-day interval (aggressive retry)
  • Music theory: major scale, diatonic chord qualities, triad intervals, cadence definitions — all musically verified
  • Pedagogy: 40+ educational insight texts with historical context, song references for ear training, functional harmonic analysis
  • Audio: synthesized via Web Audio API + WebAudioFont. Challenge audio: tonic triad context, arpeggiated chords, sequential intervals
  • Piano visualization: interactive keyboard showing played keys, toggle-to-hear, mobile zoom + scroll
  • WebGL2 post-processing: bloom, Gaussian blur, chromatic aberration, scanlines, vignette

Demo Ball

Minimal reference implementation of the Simulation trait. Click to launch a bouncing ball. Score = distance traveled. Validates the ECS pipeline and demonstrates variant system with tunable parameters.

Design Decisions

  • Single cdylib+rlib crate — same code for WASM and native
  • Zero-copy framebuffer — JS reads directly from WASM linear memory
  • All f64 math — no f32 anywhere, giving 15-16 significant digits
  • Sound as JSON drain — no JS callbacks from Rust, unidirectional command flow
  • Fixed 60Hz physics — deterministic, independent of display rate
  • Simulation trait boundary — games implement setup/step/render, engine owns timing and input
  • Deferred mutation — SpawnQueue, SoundQueue, EventBus stage changes for phase boundaries
  • Snapshot-then-commit — collision.rs collects data into Vec, processes, writes back in separate loop
  • EventBus as integration point — nearly all cross-system communication flows through frame-scoped pub/sub
